# ТЗ: MotoDating — приложение знакомств в Telegram

> **Статус:** Этап 1 (MVP) реализован и работает локально в Docker.
> Деплой пока отложен (по решению — «деплоить пока не время»).
> Язык интерфейса: **украинский**. Регион запуска: **Украина** 🇺🇦.

## 1. Идея и цель

Telegram Mini App для знакомств между **девушками** и **мужчинами** на мотоциклетную тематику.
Цель — помочь людям найти друг друга, чтобы вместе кататься на мотоцикле
(«покатать» / «покататься») и/или общаться.

Формат: **Telegram Mini App** (лента анкет с лайками) + **Telegram-бот**
(точка входа, авторизация, push-уведомления о новых мэтчах).

**Ограничение: 18+** — приложение только для совершеннолетних.

---

## 2. Роли пользователей

- **Девушка** — ищет того, кто покатает.
- **Мужчина** — ищет, кого покатать.

Логика подбора: пользователь видит анкеты **противоположной роли**.
Роль выбирается один раз при регистрации (позже можно будет менять в настройках).

---

## 3. Основные сценарии (user flow)

1. **Старт.** Пользователь открывает бота → жмёт «Открыть приложение» → запускается Mini App.
2. **Авторизация.** Автоматически через Telegram (initData), без логина/пароля.
   Подтверждение возраста 18+ при первом входе.
3. **Заполнение анкеты:**
   - Фотографии (1–5 шт.)
   - Имя
   - Возраст (для проверки 18+)
   - Город
   - Роль (девушка / мужчина)
   - Описание о себе (bio)
4. **Лента анкет.** Пользователь скроллит ленту анкет противоположной роли
   и ставит 👍 лайк тем, кто понравился (без отдельной кнопки «пропустить» —
   просто листаешь дальше).
5. **Мэтч.** При взаимном лайке — оба получают уведомление в боте + видят мэтч в приложении.
6. **Общение:**
   - **MVP:** открывается прямой чат в Telegram (ссылка на профиль / @username).
   - **Этап 2:** встроенный чат внутри приложения.

## 5. Технический стек (реализовано)

Единый TypeScript-стек (общий язык фронта и бэка). Версии библиотек подобраны
под Node.js 18 на машине разработки; в Docker используется Node 20.

| Слой | Технология | Статус |
|------|------------|--------|
| Frontend (Mini App) | React + TypeScript + Vite | ✅ реализовано |
| Backend (API) | Node.js + TypeScript, Fastify 4 | ✅ реализовано |
| Бот | grammY | ✅ реализовано |
| База данных | Prisma ORM + **SQLite** (на старте) → PostgreSQL (для прод) | ✅ SQLite |
| Хранение фото | Локально на сервере (Docker volume) → S3/R2 (для прод) | ✅ локально |
| Контейнеризация | Docker + docker-compose | ✅ реализовано |
| Деплой | Vercel / Render / Neon / R2 (см. §8) | ⏸ отложено |

Структура репозитория: `frontend/`, `backend/`, `bot/` (монорепо).
Все тексты интерфейса вынесены в `frontend/src/texts.ts` (украинский) —
правки и перевод в одном месте.

## 6. Модель данных (черновик)

```
User
  id            (telegram_id)
  role          (girl | rider)
  name          string
  age           int          (>= 18)
  city          string
  bio           text
  photos        Photo[]
  is_visible    bool
  created_at

Photo
  id
  user_id
  url
  order

Like
  id
  from_user_id
  to_user_id
  created_at

Match
  id
  user_a_id
  user_b_id
  created_at
```

Мэтч создаётся, когда существуют два встречных `Like` между пользователями A и B.

## 8. Хостинг и стоимость

> ⏸ **Деплой пока отложен** («деплоить пока не время»). Раздел — план на будущее.

**Решение:** на старте — бесплатный облачный стек (~$0/мес), переезд на VPS при росте.

| Компонент | Сервис (бесплатный тариф) |
|-----------|---------------------------|
| Frontend (Mini App) | Vercel / Cloudflare Pages (HTTPS из коробки) |
| Backend (API) | Render (Web Service, 750 ч/мес, «засыпает» при простое) |
| База данных | Neon / Supabase (Postgres, ~0.5 ГБ) |
| Хранилище фото | Cloudflare R2 (10 ГБ) / Supabase Storage |
| Бот | BotFather + webhook (бесплатно) |
| Домен | поддомены сервисов (`*.vercel.app`) |

**Переезд на платный (~$6–15/мес)** понадобится, когда: надоест «засыпание»
backend, БД перерастёт 0.5 ГБ или фото перерастут 10 ГБ → тогда VPS
(Hetzner / DigitalOcean) + домен (~$1/мес) + SSL бесплатно (Let's Encrypt).

## 9. Монетизация (идеи на будущее)

> ⏸ Пока **не внедряем** (по решению — сначала набираем аудиторию, потом деньги).
> Раздел — копилка идей.

**Принцип:** дейтинг живёт за счёт количества анкет. Пока пользователей мало —
агрессивная монетизация убьёт рост. Сначала аудитория, потом доход.

1. **Premium-подписка:** «хто мене вподобав» (кто лайкнул), безлимитные лайки,
   фильтры (город/возраст/мотоцикл), больше фото, режим невидимки.
2. **Разовые бусты:** поднять анкету, суперлайк, спотлайт (топ ленты города).
3. **Ниша (преимущество):** партнёрки с мотошколами, арендой, магазинами
   экипировки; спонсорские анкеты/баннеры мото-брендов; верификация
   («перевірений мотоцикліст»).
4. **Виртуальные подарки** внутри чата (за Telegram Stars).
5. **Платные ивенты:** совместные покатушки, мото-вечеринки (продажа билетов).

**Приём оплаты:** Telegram Stars ⭐ (цифровые товары: подписки, бусты, подарки)
+ Telegram Payments через провайдера (Fondy / LiqPay / Monobank) для билетов/реальных платежей.

**Рекомендованный порядок запуска:** аудитория бесплатно → «хто вподобав» + бусты
(Telegram Stars) → партнёрки с мото-бизнесом → подписка и ивенты.
