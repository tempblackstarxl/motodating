# 🆓 Бесплатный деплой MotoDating (показать друзьям онлайн)

Цель: за ~20 минут получить публичную ссылку, чтобы друзья открыли приложение
в Telegram. Без своего сервера и без оплаты.

Используем режим **«всё в одном»**: один сервис раздаёт и фронтенд, и API, и бота
(корневой `Dockerfile`). Значит, деплоим **один сервис** — это просто.

Стек: **Render** (хостинг, free) + опц. **Neon** (Postgres, free) + **Telegram**.

---

## Что понадобится
- Аккаунт на [GitHub](https://github.com) (код деплоится оттуда)
- Аккаунт на [Render](https://render.com) (вход через GitHub)
- Бот в Telegram (создадим в @BotFather)

---

## Шаг 1. Залить код на GitHub
Если репозитория ещё нет:
```bash
cd /Users/mrblack/motodating
git init
git add .
git commit -m "MotoDating MVP"
```
Создай пустой репозиторий на GitHub и запушь:
```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/motodating.git
git branch -M main
git push -u origin main
```

> Файлы `.env` не попадут в репозиторий (они в `.gitignore`) — это правильно,
> секреты задаём прямо на Render.

---

## Шаг 2. Создать бота
1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot`.
2. Задай имя и username (заканчивается на `bot`).
3. Сохрани **токен** (вида `123456:ABC-DEF...`) — пригодится на шаге 4.

---

## Шаг 3. (Рекомендуется) Бесплатная база Neon
Без этого шага данные (анкеты) будут **сбрасываться** при перезапуске сервиса.
С Neon анкеты сохраняются.

1. Зарегистрируйся на [neon.tech](https://neon.tech) (free).
2. Создай проект → скопируй **Connection string** (вида
   `postgresql://user:pass@ep-xxx.neon.tech/dbname?sslmode=require`).
3. В файле `backend/prisma/schema.prisma` поменяй провайдер на Postgres:
   ```prisma
   datasource db {
     provider = "postgresql"
     url      = env("DATABASE_URL")
   }
   ```
   Закоммить и запушь:
   ```bash
   git add backend/prisma/schema.prisma && git commit -m "use postgres" && git push
   ```

> Хочешь совсем по-быстрому, без Neon? Пропусти этот шаг — будет SQLite,
> но анкеты иногда будут пропадать (для короткого показа друзьям ок).

---

## Шаг 4. Деплой на Render
1. На [dashboard.render.com](https://dashboard.render.com) → **New +** → **Web Service**.
2. Подключи свой GitHub-репозиторий `motodating`.
3. Настройки:
   - **Name**: например `motodating` → твой адрес будет `https://motodating.onrender.com`
     (запомни его — это `WEBAPP_URL`).
   - **Region**: Frankfurt (ближе к Украине).
   - **Runtime / Language**: Docker (Render сам найдёт корневой `Dockerfile`).
   - **Instance Type**: **Free**.
4. **Environment Variables** (раздел Advanced / Environment) — добавь:

   | Key | Value |
   |-----|-------|
   | `DATABASE_URL` | строка из Neon (шаг 3). Если без Neon: `file:/data/dev.db` |
   | `BOT_TOKEN` | токен из @BotFather (шаг 2) |
   | `WEBAPP_URL` | `https://motodating.onrender.com` (твой адрес из п.3) |

   > `PORT` задавать не нужно — Render передаёт его сам, приложение это учитывает.

5. Нажми **Create Web Service**. Подожди сборку (несколько минут).
6. Когда статус станет **Live** — открой `https://motodating.onrender.com` в браузере,
   должно появиться приложение.

---

## Шаг 5. Подключить приложение к боту
Чтобы кнопка открывала Mini App:
1. @BotFather → `/setmenubutton` → выбери своего бота.
2. Введи URL: `https://motodating.onrender.com`
3. Введи текст кнопки, например `🏍️ Відкрити`.

Теперь у бота внизу есть кнопка, открывающая приложение. Также работает `/start`.

---

## Шаг 6. Показать друзьям
Просто пришли друзьям ссылку на бота: `https://t.me/ВАШ_БОТ_username`.
Они жмут **Start** → открывают приложение → заполняют анкету → листают ленту → лайкают.
При взаимном лайке — мэтч и кнопка «Написати».

---

## ⚠️ Важно про бесплатный тариф Render
- **«Засыпание»**: после ~15 минут без активности сервис останавливается.
  Первое открытие потом — медленное (30–60 сек), и бот может не ответить, пока
  не «проснётся».
- **Как держать бодрым** (на время показа): зайди на [cron-job.org](https://cron-job.org)
  (бесплатно) и настрой пинг `https://motodating.onrender.com/api/health` каждые 10 минут.
- **Фото**: загруженные фото лежат на временном диске и **пропадут при перезапуске**
  сервиса. Для короткого показа это ок; для постоянной работы фото надо вынести в
  облачное хранилище (Cloudflare R2) — это отдельная доработка.
- **Данные анкет**: сохраняются, только если подключил Neon (шаг 3). На SQLite —
  сбрасываются при перезапуске.

## Обновление версии
После изменений в коде:
```bash
git add . && git commit -m "update" && git push
```
Render пересоберёт и задеплоит автоматически.

---

## Если что-то не так
- **Приложение не открывается в Telegram** — Telegram требует HTTPS. Адрес
  `onrender.com` уже на HTTPS, проверь, что в @BotFather указан именно он.
- **Бот не отвечает на /start** — проверь, что в Render заданы и `BOT_TOKEN`,
  и `WEBAPP_URL`; посмотри логи в Render (вкладка Logs). Возможно, сервис «уснул» —
  открой ссылку в браузере, чтобы разбудить.
- **«Двойной» бот / конфликт getUpdates** — нельзя запускать бота с тем же токеном
  ещё где-то (например, локально `npm run dev:bot`). Один токен — один запущенный бот.
- **Города не ищутся** — нужен доступ в интернет (Render его даёт); подсказки идут
  через OpenStreetMap.

## Альтернативные бесплатные платформы
Тот же корневой `Dockerfile` подойдёт и для:
- **Railway** (railway.app) — очень просто, есть стартовый бесплатный кредит,
  поддерживает постоянные тома (можно хранить и фото).
- **Koyeb** (koyeb.com) — есть небольшой постоянный бесплатный инстанс (без «засыпания»).
- **Fly.io** — Docker + тома, чуть сложнее в настройке.

Принцип везде одинаковый: подключить репозиторий, выбрать Docker, задать
переменные `DATABASE_URL`, `BOT_TOKEN`, `WEBAPP_URL`.
